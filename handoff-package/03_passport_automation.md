## Цель

Автоматизировать первичную обработку заявки клиента: принять контактные данные и описание задачи, оценить примерный бюджет через AI, создать заказ в Bpium CRM, создать платеж YooKassa и уведомить ответственного менеджера в Telegram.

## Как запускается

Сценарий запускается через форму n8n `Форма заявки`.

Пользователь заполняет поля:

| Поле | Назначение |
|---|---|
| `name` | имя клиента |
| `email` | email клиента |
| `phone` | телефон клиента |
| `request` | описание задачи клиента |

## Что происходит на выходе

После успешного выполнения:

| Результат | Где появляется |
|---|---|
| Заявка клиента | Bpium CRM, каталог заказов |
| Оценка бюджета и короткое КП | Bpium CRM, поля заказа |
| Платежная ссылка и ID платежа | YooKassa и Bpium CRM |
| Уведомление менеджеру | Telegram |
| Проверка созданного заказа | финальная HTTP-нода `GET Check Bpium Order` |

## Ограничения текущей версии

Сценарий подготовлен как тестовый. Для production нужны отдельные production credentials, отдельные env-переменные и проверка боевых аккаунтов Bpium, YooKassa и Telegram.
