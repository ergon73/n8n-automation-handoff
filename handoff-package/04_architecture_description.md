# Архитектура сценария

## Скриншот workflow

![Схема workflow](02_architecture_workflow_screenshot.png)

## Общая логика

Сценарий принимает заявку через форму, минимизирует данные перед отправкой в AI, получает оценку бюджета, создает заказ в Bpium, создает платеж YooKassa, записывает платежную ссылку обратно в Bpium и отправляет менеджеру безопасное Telegram-уведомление без персональных данных клиента.

## Поток данных

```text
On form submission
-> SEC_Minimize_For_AI
-> AI Request
-> Parse AI
-> Create Order In Bpium
-> Create YooKassa Payment
-> Update Bpium Order Payment Link
-> Send Telegram Message
-> GET Check Bpium Order
```

## Ключевые ноды

| Нода | Назначение | Какие данные обрабатывает |
|---|---|---|
| `On form submission` | Принимает заявку через форму | имя, email, телефон, текст заявки |
| `SEC_Minimize_For_AI` | Минимизирует данные перед AI | оставляет только `request_id`, `environment`, `request_text` |
| `AI Request` | Отправляет текст заявки в ProxyAPI/OpenAI | только обезличенный текст заявки |
| `Parse AI` | Парсит JSON-ответ модели | `price`, `offer` |
| `Create Order In Bpium` | Создает заказ в CRM | контактные данные клиента, текст заявки, оценка, КП |
| `Create YooKassa Payment` | Создает платеж | сумма, ID заказа, metadata |
| `Update Bpium Order Payment Link` | Записывает платеж в CRM | ссылка оплаты, ID платежа, статус |
| `Send Telegram Message` | Уведомляет менеджера | ID заказа, ID платежа, сумма, статус, без email/телефона |
| `GET Check Bpium Order` | Проверяет созданный заказ | ID заказа |

## Где происходит обработка данных

| Точка обработки | Что важно |
|---|---|
| Входная форма | Здесь появляются персональные данные клиента |
| `SEC_Minimize_For_AI` | Главная точка защиты: AI не получает email и телефон |
| `Parse AI` | JSON от модели превращается в структурированные поля |
| Bpium-ноды | Полные данные клиента сохраняются только в CRM |
| Telegram-нода | Во внешний мессенджер уходит только минимальное уведомление |

## Нормальный успешный результат

Успешное выполнение видно в `Executions`: все ноды зеленые, Telegram output содержит `ok: true`, в Bpium создан заказ, а в YooKassa создан платеж.

## Архитектурные решения

| Решение | Зачем сделано |
|---|---|
| Отдельная нода `SEC_Minimize_For_AI` | Чтобы не отправлять персональные данные во внешний AI API |
| ProxyAPI через Credential | Чтобы токен не лежал в параметрах workflow |
| Telegram chat id через env | Чтобы не хранить маршрут уведомлений как hardcoded fallback |
| Отдельное имя `[TEST]` | Чтобы не перепутать тестовый и боевой сценарий |
| Финальная проверка Bpium | Чтобы убедиться, что заказ действительно создан и доступен по API |
