# Регламент поддержки

## Каналы обращения

| Канал | Для чего |
|---|---|
| рабочий чат проекта | обычные вопросы, уточнения, некритичные ошибки |
| личное сообщение ответственному | срочные инциденты |
| скриншот + время ошибки | обязательное приложение к любому техническому обращению |

## Уровни инцидентов

| Уровень | Пример | Реакция |
|---|---|---|
| Критичный | форма не работает, заявки не создаются, платежи не создаются | до 1 рабочего часа |
| Высокий | заказ создан, но платеж или Telegram не отработали | до 4 рабочих часов |
| Средний | некорректная сумма, ошибка в тексте КП, отдельный сбой AI | до 1 рабочего дня |
| Низкий | нужна правка текста уведомления, документации или небольшое улучшение | планово, до 3 рабочих дней |

## Кто отвечает

| Зона | Ответственный |
|---|---|
| n8n workflow | технический специалист по n8n |
| Bpium CRM | владелец CRM / технический специалист |
| YooKassa | владелец магазина / бухгалтерия / технический специалист |
| Telegram bot | владелец Telegram bot credential |
| бизнес-правила оценки | владелец процесса продаж |

## Что должно быть в сообщении об ошибке

| Поле | Пример |
|---|---|
| время | `2026-04-28 15:20` |
| кто заметил | `менеджер продаж` |
| действие | `отправили заявку через форму` |
| ожидаемый результат | `должен создаться заказ и прийти Telegram` |
| фактический результат | `заказ есть, Telegram нет` |
| ID заказа/платежа | `order_id=9`, если есть |
| скрин | ошибка в n8n, CRM, Telegram или форме |

## Эскалация

| Условие | Куда эскалировать |
|---|---|
| повторяются ошибки платежей | владелец YooKassa + n8n-специалист |
| массово не создаются заказы | n8n-специалист + владелец Bpium |
| утечка или подозрение на утечку токена | владелец инстанции n8n, немедленная ротация credentials |
| Telegram не работает, но CRM и платежи работают | можно временно работать из CRM, Telegram чинить как высокий, но не критичный инцидент |

## Что входит в поддержку

Входит:

| Работа | Пример |
|---|---|
| диагностика failed executions | найти красную ноду и причину |
| восстановление credentials/env | заменить истекший токен или cookie |
| мелкие правки текста уведомлений | изменить формулировку Telegram |
| обновление документации | записать изменения в пакет передачи |

Не входит без отдельной задачи:

| Работа | Почему отдельно |
|---|---|
| изменение бизнес-логики оценки | влияет на AI prompt и результаты продаж |
| переход с тестовой YooKassa на production | требует отдельной проверки платежей |
| замена Bpium на другую CRM | меняет архитектуру |
| добавление новых каналов уведомлений | требует новой интеграции и тестов |
